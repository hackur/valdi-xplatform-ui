# No Emoji Policy

## Usage
Reference this to ensure all code and documentation follows the no-emoji policy.

## Policy
**NEVER use emojis in code, documentation, comments, or commit messages unless explicitly instructed by the user.**

## Rules

### Code Files (TypeScript, JavaScript)
- NO emojis in comments
- NO emojis in string literals
- NO emojis in console logs
- NO emojis in error messages
- NO emojis in user-facing text

```typescript
// WRONG - Contains emoji
console.log('Starting app...');
const greeting = 'Hello world!';
throw new Error('Failed to connect');

// CORRECT - No emojis
console.log('Starting app...');
const greeting = 'Hello world!';
throw new Error('Failed to connect');
```

### Documentation Files (Markdown)
- NO emojis in headings
- NO emojis in lists
- NO emojis in tables
- NO emojis as bullet points
- Use standard markdown formatting

```markdown
WRONG:
# Features
- Fast performance
- Easy to use
- Great DX

CORRECT:
# Features
- Fast performance
- Easy to use
- Great developer experience
```

### Commit Messages
- NO emojis in commit titles
- NO emojis in commit descriptions
- Use conventional commit format

```
WRONG:
feat: Add chat feature

CORRECT:
feat: Add chat feature
```

### Alternatives to Emojis

Instead of emojis, use:

**Lists and bullet points:**
```markdown
## Key Features
- Feature 1
- Feature 2
- Feature 3
```

**Tables for structured information:**
```markdown
| Feature | Status | Notes |
|---------|--------|-------|
| Chat    | Done   | Fully implemented |
| Voice   | WIP    | In progress |
```

**Diagrams for architecture:**
```markdown
## Architecture
```
Module A --> Module B --> Module C
```
```

**Headings and subheadings:**
```markdown
## Important
This is a critical section.

## Warning
Be careful with this configuration.
```

**Status indicators with text:**
```markdown
Status: COMPLETE
Status: IN PROGRESS
Status: BLOCKED
```

### No Authorship or Signatures
- NO author names in files
- NO "Generated by..." comments
- NO company names or product names
- NO model names or versions
- NO timestamps or signatures

```markdown
WRONG:
## Component Documentation
Author: John Doe
Generated with Tool X
Created: 2024-01-15

CORRECT:
## Component Documentation
```

```typescript
// WRONG
/**
 * ChatService
 * Author: Jane Smith
 * Created with AI Assistant
 */

// CORRECT
/**
 * ChatService
 * Handles chat API communication
 */
```

### Exception
Emojis are ONLY allowed when:
1. The user explicitly requests emoji usage
2. The emoji is part of user-facing content and explicitly specified
3. The emoji is in test data or fixtures for testing emoji handling

## Verification
Before committing, search for emoji patterns:
```bash
# Search for common emoji patterns
grep -r "[\u1F600-\u1F64F]" .claude/
grep -r "[\u2600-\u26FF]" .claude/
```

## Remember
- Professional tone
- Clear, descriptive text
- Structured formatting
- No decorative elements
- No attribution
- No branding
